(схема)
- -> (решение о вмешательстве в проблемную ситуацию, потребности субъекта, стейкхолдеры, аналитики) исследование потребности
- -> (облако проблем стейкхолдеров, аналитики) выявить проблематику
- -> (дерево проблем, проблематика, аналитики, виды ресурсов) выявить проблему из среды 
- -> (комплекс моделей) исследовать ресурсные возможности
- -> (порядок объема необходимых ресурсов) построение желаемого состояния to be -> (модель желаемого состояния)

На текущем этапе выявляется количественная и качественная информация, относящаяся к проблеме, определяется к какому классу относится рассматриваемая проблема (хорошо структр, слабо структр, неструктр)
Переход от концептуальной модели к элементной, с определением структуры и границ со средой.

Шаг 1. Исследование потребностей
Уточнение всех субъектов, имеющих отношение к проблеме, анализ их интересов. Выявление облака проблем стейкхолдеров.
Другими словами, получить субъективные оценки проблемы стейкхолдерами.
Шаг связан с тем фактом, что решение проблемы без учета потребностей заинтересованных лиц может привести к решению, которое породит множество новых проблем.

Шаг 2. Выявление проблематики
Проблематика - совокупность проблем, связанных с исследуемой областью, без учета которых она не может быть решена. 
Первоначальная проблема - ядро, вокруг которого располагаются проблемы стейкхолдеров.  
Далее необходимо провести структуризацию проблем, построить их иерархию по важности, строить путем согласования с стейкхолдерами

Шаг 3. Выделение проблемы из среды
Реализация шага подразумевает применение общей схемы системного подхода, включающей аспекты:
- элементный
- структурный
- функциональный
- исторический
К моменту уже есть краткая формулировка исходной проблемы (концептуальная модель), проблематика и дерево проблем, необходимо построить количественную модель проблемной ситуации
- Стадия 1. Концептуальный сценарий проблемной ситуации
Вся имеющая информация о ситуации излагается в виде связанного текста, прочитав который субъект может понять о чем идет речь и что относится к проблемной ситуации
- Стадия 2. Классификация элементов
В итоге имеется список понятий и объектов, имеющих отношение к проблеме. Для прояснения ситуации необходимо провести группировку элементов системы по классам общности
Вариант классификации - управляемые и неуправляемые (т.е. может ли субъект на них воздействовать)
К ограничениям относим граничные условия - предельные значения характеристик элементов
Классификация всех элементов ситуации позволяет упорядочить информацию и облегчает определение структуры проблемы
- Стадия 3. Выявление структуры
Структура - совокупность взаимосвязей между элементами. В реальных задачах системного анализа данный этап довольно сложен, так как аналитик сталкивается с многоструктурностью. 
Взаимосвязи между элементами системы отражают взаимообмены ресурсами, для каждого вида ресурсов можно построить свою структуру системы.
Аналитик должен определить, какие виды ресурсов будут определять структуру системы, только затем определять соответствующие структуры.
- Стадия 4. Определение функций
Естественное продолжение этапа определения структуры. Следует охарактеризовать связи между элементами более детально - функции каждого элемента. Подобрав модель, связывающую вход и выход элемента системы, можно превратить его из черного ящика в прозрачный.
- Стадия 5. Модель as is
К данному моменту имеется достаточно детальная модель системы. Для более полного уяснения проблемы требуется определить ее место в иерархии систем окружающей среды, в какой надсистеме данная проблема будет выступать в качестве элемента.
Такое рассмотрение позволяет уточнить описание системы и ее границы. Полученная картина позволяет более четко определеить какова желаемая ситуация to be.

Шаг 4. Исследование ресурсных возможностей
Определить какие масштабы ресурсов могут понадобиться

Шаг 5. Построение желаемого состояния to be
Поскольку уже имеется определенная система as is, то возникает желание определить модель желаемого состояния.
