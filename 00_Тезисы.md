### 3. Дихотомический поиск
На каждой итерации вычисляем вда значения ЦФ функции в точках, отстоящих на eps в одну и другую сторону от середины текущего интервала

0 < 2eps <= l

Алгоритм:
- задаем eps, l, a1,b1
- если bk - ak <= l, конец
- Иначе вычисляем lambda, mu
- Если f(lambda) <= f(mu), то ak+1 = ak, bk+1 = mu
- Иначе ak+1 = lambda, muk+1 = muk
- k+=1 и снова

### 4. Метод золотого сечения
Для сокращения интервала используем одну из внутренних точек предыдущей итерации
alpha = 0,618
lambda = ak + (1-alpha)(bk-ak)
mu = ak + alpha(bk-ak)

Алгоритм:
- задать l, a1,b1
- Если bk-ak <= l, то конец
- вычислить lambdak, muk
- Если f(lambdak) <= f(muk), то
	- bk+1 = muk, muk+1 = lambdak, ak+1 = ak
	- вычислить lambdak+1
- Иначе
	- ak+1 = lambdak, lambdak+1 = muk, bk+1=bk
	- Вычислить muk+1

### 5. Метод Фибоначчи
Аналогичен методу золотого сечения, но коэф сокращения интервала меняется на итерациях

кол-во итераций - Fn > (b1-a1) / l

lambda = ak + Fn-k-1 / Fn-k+1 (bk-ak)
mu = ak + Fn-k / Fn-k+1 (bk-ak)

На первой итерации два вычисления функции, на последующих - 1

Алгоритм:
- Задать eps > 0, l > 0, a1,b1
- Выбрать Fn > b1-a1 / l, вычислить lambda, mu
- Если f(lambda) <= f(mu)
	- ak+1 = ak, bk+1 = muk, muk+1 = lambdak, lambdak+1 = ak+1 + Fn-k-2 / Fn-k (bk+1 - ak+1)
- Иначе ak+1 = lambdak, bk+1 = bk, lambdak+1 = muk, muk+1 = ak+1 + Fn-k-1 / Fn-k (bk+1 - ak+1)
- Если k = n-2:
	- lambdak = lambdak-1, muk = muk + eps
	- Если f(lambdak) > f(muk), то an = lambda

### 6. Методы одномерной оптимизации с использованием информации о производных
- Метод деления отрезка пополам
Применяется к псевдовыпуклым дифф функциям
Вычисляется производная функции в середине отрезка и на этой основе интервал сокращается. 
- Если производная >0, то min лежит слева от точки
- Если производная <0, то min лежит справа от точки

Алгоритм:
- задаем a1,b1, l
- положить n - число для которого (1/2)^n <= l/(b1-a1)
- положить lambda = 1/2(ak+bk) и вычислить f(lambda)
- если производная = 0, то конец
- Если производная > 0:
	- ak+1 = ak, bk+1 = lambda
- Если производная < 0:
	- bk+1 = bk, ak+1 = lambda
- Если k = n, то конец, иначе k += 1


- Метод Ньюьтона
Основан на квадратичной аппроксимации функции в заданной точке lambda
Начальная точка выбирается вблизи значения искомого оптимума, обеспечивая хорошую сходимость.
R(lambda) = f(lambda) + f'(lambdak)(lambda-lambdak)+ 1/2f''(lambdak)(lambda-lambdak)^2
В качестве следующей точки берется точка в которой R'(lambdak+1) = 0

lambdak+1 = lambdak - (f'(lambdak))/(f''(lambdak))
Процесс останавливается, когда |lambdak+1 - lambdak| < eps

### 7.  Метод случайного поиска
Суть в генерации случайных точек, равномерно распределенных на [a,b] и фиксации точки, соотв наименьшему значению ЦФ
- Задаем eps, требуемую вероятность p
- рассчитываем кол-во итераций n >= ln(1-p)/ln(1-(2eps/b-a))
- Генерируем случайную величину lambdak, равномерно распределенную на [a,b] lambdak = a + X(b-a), X - случайная величина на [0,1]
- вычисляем f(lambdak)
- если f минимальна, то запонимаем, повторяем n раз
- С вероятностью p минимум лежит на отрезке [lambda* - eps, lambda* +eps]

### 9. Метод Гаусса-Зейделя
Осуществляется поиск вдоль направлений d1, d2, .. dn, где dj - вектор, все компоненты которого за исключением jго равны нулю
При поиске по направлению dj, меняется только переменная xj, а остальные остаются зафиксированными
Конец, если ||xk+1 -xk|| < eps

Алгоритм:
- Задать eps, x1 - начальная точка
- положить y1 = x1, k = j = 1
- Решить задачу min f(yj + lambda dj), положить yj+1 = yj + lambda dj
- если j < n, j+=1, пройти снова
- если j=n, xk=1 = yj+1, если норма < eps, то конец, иначе положить y1 = xk+1
